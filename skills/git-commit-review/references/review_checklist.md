# 代码审查检查清单

## 基础检查项

### 1. 代码质量
- [ ] 代码风格是否一致
- [ ] 变量和函数命名是否清晰
- [ ] 代码复杂度是否合理
- [ ] 是否存在重复代码
- [ ] 注释是否充分且准确

### 2. 功能正确性
- [ ] 修改是否解决了预期问题
- [ ] 边界条件是否处理正确
- [ ] 错误处理是否完善
- [ ] 逻辑是否清晰正确

### 3. 性能考虑
- [ ] 是否存在性能瓶颈
- [ ] 数据库查询是否优化
- [ ] 内存使用是否合理
- [ ] 算法复杂度是否适当

### 4. 安全性
- [ ] 输入验证是否充分
- [ ] 是否存在安全漏洞
- [ ] 敏感信息是否妥善处理
- [ ] 权限控制是否正确

## 特定类型检查

### 数据库相关
- [ ] SQL 注入防护
- [ ] 索引使用是否合理
- [ ] 事务处理是否正确
- [ ] 数据迁移是否安全

### API 变更
- [ ] 向后兼容性
- [ ] API 文档更新
- [ ] 版本控制策略
- [ ] 错误响应格式

### 配置文件
- [ ] 敏感信息是否加密
- [ ] 环境特定配置
- [ ] 默认值是否合理
- [ ] 配置验证逻辑

### 前端代码
- [ ] 用户体验影响
- [ ] 浏览器兼容性
- [ ] 响应式设计
- [ ] 无障碍性考虑

## 影响范围评估

### 低风险变更
- 代码注释和文档
- 单元测试添加
- 日志输出调整
- 代码格式化

### 中风险变更
- 业务逻辑修改
- 新功能添加
- 依赖库更新
- 配置参数调整

### 高风险变更
- 核心算法修改
- 数据库结构变更
- 安全相关修改
- 外部接口变更

## 建议评估标准

### 必须修改 (Critical)
- 安全漏洞
- 功能错误
- 性能严重问题
- 违反编码规范

### 建议修改 (Major)
- 代码可读性问题
- 潜在性能问题
- 测试覆盖不足
- 文档缺失

### 可选修改 (Minor)
- 代码风格细节
- 变量命名优化
- 注释完善
- 小幅性能优化
